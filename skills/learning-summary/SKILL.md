# 学習サマリー HTML 生成スキル

今の学習セッション（会話）で **何を・どんな順で学び、何を習得したか** を、
読み返しやすいビジュアルな1枚のHTMLにまとめ、`learning-log/<日付>.html` に保存する。

## 重要な原則
- **捏造しない。** 実際にこのセッションで起きたこと・本人が自分で辿り着いた結論だけを書く。これは記録物。
- HTML/CSS は学習の本筋ではなく "足場" なので、こちらで完成させてよい。
- 日本語・1枚もの・**外部依存ゼロ**（そのままブラウザで開ける）。

## 手順
1. **このスキルフォルダの `template.html` を読む。** デザイン（CSS・モーダル・スクリプト）はこれを丸ごと使い、毎回の見た目をそろえる。
2. 今の会話を振り返り、次を抽出する：
   - セッションの**テーマ（タイトル）**と**今日の日付**
   - **学びの流れ**：順番に並ぶ「ステップ」。各ステップ＝短い見出し＋1行概要＋「なぜ次に進んだか」の一言。
     さらに各ステップの**詳細**（困りごと／具体例・解決／たとえ／気づき）。
   - **習得したキー概念**：用語＋平易な定義。
   - **通し例**：題材にした具体例。
   - 本人が書いたコードや図など、**具体的な成果物**（あれば）。
   - **ワナ・実務の勘所**（あれば）。
   - **到達点の一文**（「後輩に説明するなら」的な集大成）。
   - **次回の入口**。
3. `template.html` の `<!-- FILL: ... -->` と「例」コメントを、抽出した内容で置き換える：
   - 「学びの流れ」は**ステップの数だけ** `.step`（`data-step="1..N"`）と、対応する `.detail-content`（`data-detail="1..N"`）を生成。**番号は必ず一致**させる。
   - 詳細ブロックの色分けは固定：`problem`(困りごと)／`example`(具体例・解決)／`analogy`(たとえ)／`insight`(気づき)。
   - そのセッションに無いセクション（例：図がない）は丸ごと削ってよい。**例・コメント・ダミーは残さない。**
4. `learning-log/<YYYY-MM-DD>.html` に保存（フォルダが無ければ作る）。同名があれば上書き前に確認するか `-2` 等を付ける。
5. 保存先パスを伝え、調整希望（文章量・色・「自分の言葉メモ」欄の追加など）を聞く。

## 出力の構成（template.html の順序）
ヒーロー → ひとことまとめ → 学びの流れ（タップで詳細モーダル）→ キー概念カード →
通し例 →（任意）図・成果物 → ワナ → 到達点の一文 → 次回の入口 → フッター。
