習得したキー概念

今日の主役

過学習（overfitting）

訓練データを丸暗記して、新しいデータで失敗する状態。train↑ / test↓ の「差」で見抜く。

ML の目的

汎化（generalization）

見ていないデータでも通用する、本物の賢さ。Kaggle で評価されるのはこちら。

今日の道具

train / test split

正解付きデータの一部を隠し、学習に使わず採点だけに使う作法。Kaggle が答えを隠す構図と同じ。

役割の整理

検査 vs 治療

過学習を「見つける」のが train/test split（検査）、「治す」のが複雑さを下げる＝深さを浅くする（治療）。

最適点

ちょうどいい複雑さ

浅すぎ＝表現不足、深すぎ＝丸暗記。test が頂点になる所が最適（今回は深さ3）。

純度の限界

矛盾するデータ

手がかりが完全一致なのに答えが割れる行（今回8組）があると、完全分離は原理的に不可能。これが純度の天井。

抜き打ち試験の結果（深い木 vs 浅い木）

深い木（制限なし・深さ14・葉114枚）
train 98.7% → test 73.0%　差 +25.7pt ＝ 丸暗記（過学習）
浅い木（max_depth=3・葉8枚）
train 80.7% → test 84.0%　差 −3.3pt ＝ 汎化（本物）

過学習曲線：深さ 1→15 で成績を追うと、train（見たデータ）は深くするほど上がり続け 98.7% に張り付く。一方 test（抜き打ち）は深さ3で頂点 84% → 以降 73% 前後へ低迷。2本の線の開きが広がっていくのが、過学習が起きている姿。test の本当の勝者は 浅い木（84% > 73%） ── 予想どおり。

途中で踏んだ "ワナ"（実務の勘所）

「深い＝賢い」の罠：直感に反して、複雑にするほど汎化は壊れる。ちょうどいい複雑さに頂点があり、それを超えると賢さは下がる。
訓練成績の罠：「見たデータへの精度」は丸暗記でいくらでも盛れる（深い木の98.7%）。評価は必ず見ていないデータで行う。
test の見すぎの罠（予告）：test を見ながら深さを何度も選ぶと、今度は test に過学習してしまう。これを防ぐのが validation / 交差検証 ── 森の物語の後で回収。

1タイトル

昨日 max_depth=3 と書いた「3」を、誰が決めたのか振り返った。

具体例

「3」は読みやすさで私が提案しただけ。8 でもよかったし、max_depth=None（上限なし）で、葉がほぼ一色になるまで育てることもできる。

気づき

ならば「制限を外して深く育てた方が、細かく見分けられて賢いのでは？」という素朴な疑問が湧く。今日はこの直感を叩く。

手を動かす前に、制限なしで育てたら訓練データの正解率がどうなるか予想した。

予想

「完全には分けられず、純度の限界が来そう」── 100% にはならず頭打ちになるはず、という読み。

気づき

「限界がある」という直感は、このあと実データで当たることになる。

制限なし（max_depth=None）で実際に育ててみた。

具体例

深さ 14・葉 114 枚・訓練データ正解率 98.0%。100% にならない理由は、手がかりが完全一致なのに答えが割れる「矛盾」が 8 組あるから（入力が同じなら、どんな if 文の木でも区別しようがない）。

気づき

予想どおり純度には限界があった。でも 98% は昨日の 82% より高い ── 一見「深い木の勝ち」に見える。

98% という数字が「何のデータへの成績か」に引っかかった。

たとえ

解答集を手に持って模試を受けた生徒は、当然 98 点を取れる。さて ── それだけを根拠に、その生徒が賢いと言えるか？

気づき

深さ 14・葉 114 の木は、400 件をほぼ 1 件ずつ仕分ける巨大なルックアップ表＝丸暗記。既知の入力には完璧でも、それは賢さではない。

本当の賢さは「見ていないデータ」で測る、と気づいた。

具体例

正解付きデータを train 300 件 / test 100 件 に分割（train_test_split）。train だけで学習し、test は絶対に見せず採点だけに使う。

たとえ

Kaggle がテストの正解を隠すのと同じ。隠すからこそ、丸暗記でカンニングできない。

気づき

「答えを全部使わない」ではなく「答えの一部を採点用に隠す」。この分離の重要性を自力で言い当てた。検査（split で見つける）と治療（複雑さを下げる）は別の役割。

train/test に割って採点し直し、予想と突き合わせた。

具体例

深い木：train 98.7% → test 73.0%（差 +25.7pt）。浅い木：train 80.7% → test 84.0%（差 −3.3pt）。test の成績は 浅い木 84% > 深い木 73%。

気づき

「浅い木の方が汎用的だから勝つ」という予想が的中。深い木の 98.7% は、抜き打ちでは 73% まで落ち、簡単な木より下だった。train と test の 25pt の開きが、過学習が見える瞬間。

深さ 1→15 で、train と test の成績曲線を描いた。

具体例

train（見たデータ）は深くするほど上がり続け 98.7% に張り付く。test（抜き打ち）は深さ 3 で頂点 84% → 以降 73% へ低迷。2 本の線の開きが広がっていく＝過学習。

気づき

浅すぎ＝表現不足、深すぎ＝丸暗記。木 1 本では「賢さ」と「安定」を両立しにくい。これが Day 1 の予言「木 1 本は弱い」の正体で、次の森の物語（ランダムフォレスト → GBDT / LightGBM）を呼ぶ。
